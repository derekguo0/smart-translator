# 上下文信息使用指南 - 让翻译更精准

## 🎯 快速上手

### 第一步：识别内容类型
在开始翻译前，先判断您的文本属于什么类型：

| 内容类型 | 特征 | 推荐上下文 |
|----------|------|------------|
| 技术文档 | 包含API、代码、技术术语 | `技术文档` |
| 商务邮件 | 正式沟通、客户对话 | `商务邮件` |
| 学术论文 | 研究数据、学术用词 | `学术论文` |
| 新闻报道 | 时事、媒体用语 | `新闻报道` |
| 法律文件 | 合同、条款、法规 | `法律文件` |
| 医疗报告 | 病症、诊断、医学术语 | `医疗报告` |

### 第二步：输入框中填写
在"上下文信息 (可选)"输入框中输入相应的关键词。

## 📋 具体输入示例

### 🔧 技术文档翻译

**输入上下文**：`技术文档`

**原文示例**：
```
The API endpoint returns a JSON response with user data. 
If the request fails, check the error code in the response headers.
```

**翻译效果对比**：

❌ **无上下文**：
```
API端点返回包含用户数据的JSON响应。
如果请求失败，请检查响应头中的错误代码。
```

✅ **有上下文**：
```
该API接口返回包含用户数据的JSON响应。
如果请求失败，请检查响应头中的错误代码。
```

## 🎨 进阶技巧

### 1. 组合使用
可以使用多个关键词组合：
- `技术文档 API接口`
- `商务邮件 客户沟通`
- `学术论文 医学研究`

### 2. 具体化描述
更具体的上下文会带来更好的效果：
- `软件开发文档` > `技术文档`
- `客户投诉邮件` > `商务邮件`
- `临床试验报告` > `医疗报告`

### 3. 避免的输入
❌ 不要输入：
- `重要文件`（太模糊）
- `需要翻译的内容`（无意义）
- `中文翻译`（无帮助）
- `请帮我翻译`（不是上下文）

## 🔍 实际测试方法

### 测试步骤：
1. 选择一段专业文本
2. 先不填上下文，使用ChatGPT翻译
3. 再填写相应上下文，重新翻译
4. 对比两次结果的差异

### 效果评估标准：
- ✅ 专业术语更准确
- ✅ 语言风格更合适
- ✅ 表达方式更自然
- ✅ 语境理解更到位
